Potrzebujemy Ciebie!

Każdy może wnieść swój wkład w Slim Framework. Ten projekt jest open source i jest ulepszony pod każdym możliwym względem dzięki współpracownikom takim jak Ty.

Zobacz kod źródłowy
Zgłaszanie problemów z bezpieczeństwem

Jeśli uważasz, że znalazłeś problem dotyczący bezpieczeństwa w Slim Framework, nie używaj "forum wsparcia" lub narzędzia do śledzenia problemów i nie publikuj problemu publicznie. Zamiast tego wyślij wiadomosc e-mail z opisem problemu na adres security [at] slimframework.com.

Jak możesz pomóc?

Zawsze jest coś, co wymaga naprawy lub poprawy. Możesz znaleźć listę zaległych zadań na stronie GitHub Issue Tracker. Przejrzyj otwarte problemy i rozpocznij dyskusję na temat odpowiednich wątków, jeśli chcesz pomóc.

Jeśli nie wiesz, jak pisać kod, nie martw się! Możesz pomóc ulepszyć dokumentację. Możesz nawet poprawić literówki. Nie myśl, że musisz być genialnym programistą, aby pomóc. Cokolwiek zrobisz, będzie bardzo mile widziane!

Wytyczne dla współpracowników

Wszyscy programiści róznią sie od siebie i dlatego projekt Slim Framework ma wytyczne dla współpracowników. Wytyczne zapewniają to, że wszyscy współtwórcy postępują tak samo reguły i ten sam styl kodu.

  1. Pisz kod kompatybilny ze standardem PSR-2
  2. Używaj znaków przejścia do nowej linii w stylu UNIX`a
  3. Usuń końcowe białe spacje
  4. Użyj czterech spacji zamiast tabulacji

Żądanie Pull

Oddziel każdą nową funkcję lub ulepszenie do osobnej gałęzi w rozwidlonym repozytorium. Wyślij żądanie ściągnięcia z każdej gałęzi funkcji do repozytorium Slim Framework gałęzi develop . Wszystkie zmiany powinny obejmować odpowiednie testy PHPUnit.
